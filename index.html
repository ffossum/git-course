How Git works

Basic Git usage

Help!

Show built-in documentation

$ git help <command>

Init

Initialize your Git repository!

$ git init MyStuff

This creates a .git-folder in a new (or existing) MyStuff-project, containing

  • Configuration
  • All project (meta)data

Status

What's happening in my workspace?

$ git status


$ git status
On branch master

Initial commit

nothing to commit (create/copy files and use "git add" to track)

Status (cont.)

$ echo "Content" > MyFirstFile.txt


$ git status
On branch master

Initial commit

Untracked files:
  (use "git add <file>..." to include in what will be committed)

    MyFirstFile.txt

nothing added to commit but untracked files present (use "git add" to track)

Add

Adding new or changed files to the commit (staging)

$ git add MyFirstFile.txt


$ git status
On branch master

Initial commit

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

    new file:   MyFirstFile.txt

RM

Removing/deleting existing files completely

$ git rm MyExistingFile.txt


$ git status
On branch master
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

    deleted:    MyExistingFile.txt

Diff

Show differences between e.g. local file and previous commit

$ git diff
$ git diff --cached

commit

Commit changes to repository

$ git commit -m "Add support for new feature xxx"


[master a8be286] Add support for new feature xxx
1 file changed, 0 insertions(+), 0 deletions(-)
create mode 100644 SomeNewFile.txt

Checkout

Checkout a branch or paths to the working tree

May be used to switch between branches, overwrite local changes with the original or another version

$ git checkout anotherBranch

# Reset the file to HEAD
$ git checkout MyFirstFile.txt

.gitignore

File with patterns for files Git should ignore


$ cat .gitignore
# Ignore Intellij configuration files:
.idea/

# Ignore target folders from build:
target/

# Ignore property file templates:
*.properties.template

# Ignore self (don't commit, local only!)
#.gitignore

Branching

$ git branch testing

$ git checkout testing

$ git commit -m 'Made a change'

$ git checkout master

$ git commit -m "Made other changes"

Merging

$ git checkout -b iss53

$ git commit -m "iss53: Added new footer

Exercise: What is the result after the following commands? 

$ git checkout master
$ git checkout -b hotfix
$ git commit -m "Fixed broken email address"

$ git checkout master
$ git checkout -b hotfix
$ git commit -a -m "Fixed broken email address"

We want to merge changes from hotfix into master. 

$ git checkout master
$ git merge hotfix

Updating f42c576..3a0874c
Fast-forward
 README | 1 -
 1 file changed, 1 deletion(-)

$ git branch -d hotfix
$ git checkout iss53
$ git commit -a -m "iss53: Finished the new footer"

Now we want to merge changes from iss53 into master. 

$ git checkout master
$ git merge iss53

Auto-merging README
Merge made by the 'recursive' strategy.
 index.html | 1 +
1 file changed, 1 insertion(+)

Rebasing

$ git checkout master
$ git merge experiment

$ git checkout experiment
$ git rebase master
First, rewinding head to replay your work on top of it...
Applying: added staged command
$ git checkout master
$ git merge experiment

Pull

Pull changes from remote repositories

(requires that you've got no local changes to tracked files (see stash!))

$ git pull
$ git pull --rebase

(Fetch + merge does the same as 'git pull' in distinct operations, handy in some situations)

Push

Push your local changes remotely

(requires that you've pulled the latest changes in the relevant branches)
